# LLM 생성 코드 보안 취약점 분석

## 연구 개요

**주제**: 바이브 코딩(비전문가의 LLM 기반 개발) 환경에서 생성된 웹 서비스의 보안 취약점 분석

### 실험 설계

- **비교 대상 LLM**: GPT-4o / Gemini 1.5 Pro / Claude Sonnet
- **분석 기준**: OWASP Top 10 (2021)
- **실험 서비스 기능**: 회원가입·로그인, 게시판(작성/조회/삭제), 파일 첨부, 댓글
- **프롬프트 조건**
  - 조건 A: 보안 키워드 없이 기능만 요청
  - 조건 B: 동일 프롬프트 + "보안도 신경써서 만들어줘" 추가
- **총 케이스**: 6개 (3 LLM × 2 조건)

## 공부 순서

```
Week 1  OWASP 개념 + PortSwigger 랩 실습 + 도구 설치
Week 2  도구 심화 실습 (ZAP, Burp Suite, Semgrep)
Week 3  Node.js 환경 직접 구축 + 취약 코드 재현
Week 4  실험 체크리스트 & 분석 기준 표준화
```

## 논문 품질을 위한 핵심 원칙

1. **동일한 도구, 동일한 룰셋**으로 6개 케이스 스캔
2. 취약점 분류는 **OWASP CWE 번호**로 표기
3. 심각도는 **CVSS v3.1 점수**로 수치화
4. 주관적 판단 배제 — 정량적 근거 중심으로 작성
